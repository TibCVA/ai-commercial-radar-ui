User Guide

AI Commercial Radar

Dual-engine platform for Target Screening (Funding/Grants) and M&A weak signals, with Stage1 longlist, Stage2 profiling and outreach.

What is it?

AI Commercial Radar continuously scans curated sources (media, investor sites, grants datasets) to identify commercial opportunities before they become obvious. The platform now runs two engines: screening for actionable targets and weak-signal monitoring for pre-announcement M&A situations.

M&A Deals

Acquisition rumors, divestitures, partnerships, JVs, carve-outs, spin-offs - detected before the official announcement.

Key Actors

Developers, infra funds, corporates, operators - with their recent signals, pipeline and executives.

Strategic Signals

Management changes, permits, fundraising, advisory mandates, expansions - classified by confidence level.

Engine Modes

screening_targets_v1 (default)

Use this mode for target discovery and qualification.

  • Stage1: Longlist batches (10/20/30)
  • Validation: validate/discard selected targets
  • Stage2: profiling (lite|standard|deep)
  • Outreach: hook uses funding/grants + timing + source URLs

weak_signals_mna

Use this mode for pre-announcement M&A/advisory opportunities.

  • Maturity gate: pre-announcement only
  • Scores: need_probability + weak_signal_score
  • Automatic exclusion of matured signals
  • Prioritization by urgency window and confidence

1 Create a watchlist

A watchlist represents a monitoring scope. Each watchlist precisely defines what you're looking for: sector, geographies, operation types, etc. You can create as many watchlists as needed.

Access: Dashboard → New watchlist or Watchlists tab

Form fields

Field Description
Name Free name to identify the watchlist (e.g., "Infra Europe Datacenter")
Owner Person responsible for the watchlist. Choose from the list (Thibaut, Robin, Joel, Matthieu, Cornelius, Abraham) or enter a custom name via "Other". The owner is visible on all screens and allows filtering watchlists.
Sector Main search sector (Datacenters, Renewable energy, Transport, Telecom, etc.)
Sector tags Additional keywords to refine the search (e.g., "cloud", "fiber optic", "solar"). Type then press Enter to add.
Geographies Target countries/regions. Check desired areas and/or add custom geographies.
Operation types M&A rumor, asset deal, JV/partnership, greenfield, refinancing, carve-out, spin-off, take-private.
Actor types Developers, infra funds, corporates, operators, advisors, governments.
Freshness (days) Search time window. Default is 30 days. Increase for broader history.
Scan mode Manual only. Launch scans yourself from the dashboard.
Confidence threshold Filters results by minimum reliability level (see Confidence levels section).

Advanced options

Field Description
Target companies Company names to monitor as priority. The AI will search for them first.
Excluded companies Company names to ignore (e.g., already known deals or no interest).
Preferred sources Media or sites to prioritize (e.g., "Handelsblatt", "Infrastructure Investor").
Deal size (min/max) Range in euros to filter by estimated deal value.
Free instructions Free-text instructions sent directly to the LLM (e.g., "Focus on brownfield assets over 100MW").
Engine type Default: screening_targets_v1. You can switch to weak_signals_mna for pre-announcement M&A monitoring.
Screening defaults Funding window defaults: 1M-10M EUR, 6-24 months, batch size 10/20/30, source packs and sourcing strategies.
Stage1/Stage2 controls Use Stage1 batch controls, validation actions, and Stage2 profiling depth (lite|standard|deep) to qualify targets before outreach.
Weak-signal policy In weak_signals_mna, maturity gate stays active (pre_announce_only) and matured deals are excluded from active lists.

2 Run a scan

A scan launches a thorough AI search across all relevant sources for your watchlist. The process can take several minutes as the AI performs an exhaustive search.

a

Individual scan

On the dashboard, click the play button (triangle) on the watchlist card.

b

Global scan

Click "Scan all watchlists" at the top of the dashboard to launch a simultaneous scan on all your active watchlists.

Be patient: A full scan takes approximately 10 minutes. The AI executes a 5-stage pipeline to ensure quality results. Do not close the page during the scan.

Pipeline (v11 Multi-engine)

1
PLAN - query plan is generated from watchlist criteria and SourceCatalog templates.
2
SEARCH - active source packs are executed in parallel with source quality tagging.
3
TRIAGE - hard filters and maturity rules apply by engine type (screening vs weak signals).
4
EVIDENCE PACK - top evidence URLs are stored per event/target for auditability.
5
VALIDATION - outputs are written as events/targets (aliases deals/actors kept for compatibility).
Post-LLM validation (types, stages, currencies, URLs) and ISO normalization
Automatic confidence score calculation (0-100)
SHA-256 deduplication with cross-run enrichment

3 View results

Access: Results tab or click "View →" in the dashboard history table

The Results page displays two tabs and adapts automatically to the selected watchlist engine.

Default mode: screening_targets_v1. Tabs are shown as Events and Targets.

Events Tab

Funding/grant events with type, round, amount, date, beneficiary and evidence URLs. In weak mode, this tab remains deal-centric.

Targets Tab

Targets show pipeline stage, funding recap, fit/timing/priority scores, rationale, ownership view, hooks and evidence links.

Available filters

By watchlist

Select a specific watchlist or view all results.

By confidence

High, medium, or low.

By operation type

M&A rumor, asset deal, financing, growth signal, etc.

By freshness

Last 24h, 7 days or 30 days.

Need Probability

Filter deals by minimum short-term advisory/financing need probability.

Urgency

Filter by urgency window in days (default short-term target: 120 days).

Advisory Need Type

Filter by need category: financing, strategic review, carve-out, M&A, growth capital.

Maturity Status

Distinguish pre_announce_pass, matured_excluded, and other states.

Screening filters: pipeline_stage, funding_age_bucket, funding_amount_bucket, ownership, source_confidence. Available actions: Validate selection, Profile (Step2), then Generate outreach.

Weak-signal mode: cards display weak_signal_score, need_probability, top triggers and disqualification reason. Sorting stays need_probability DESC, then weak_signal_score DESC, then recency.

5 Export data

Export identified events/targets (and legacy deals/actors aliases) in CSV format (Excel compatible) for offline analysis or team sharing.

From the Results page

Select a watchlist then click the green "Export XLSX" button. The CSV file downloads automatically with all watchlist data (deals + actors).

From the dashboard

The export button (down arrow) on each watchlist card triggers the same CSV download.

Format: CSV with semicolon separator (;), UTF-8 encoding with BOM for optimal Excel compatibility. Contains two sections: deals and actors.

6 Understanding the dashboard

The dashboard is your overview. It displays the real-time status of all your watchlists.

Top counters

Total number of active watchlists, detected deals (cumulative), identified actors (cumulative), and last scan date.

Watchlist cards

Each card shows owner, engine_type, sector, geographies, last scan date, and cumulative counts (events/targets). For screening watchlists you also see candidates/validated/profiled/high-priority KPIs plus a Next batch action.

Filter by owner

Use the dropdown at the top of the list to view only watchlists from a specific owner, or select "All" to display everything.

Recent activity & history

The sidebar shows recent activity, and the bottom table summarizes scan history with status and total result count.

7 Confidence levels

Each deal and actor is accompanied by a confidence level assigned by the AI, based on the quality and quantity of sources found.

H

High

Multiple converging credible sources or direct announcement from a stakeholder. Strong, reliable signal.

E.g.: Handelsblatt article + advisor press release + Reuters confirmation

M

Medium

Multiple indirect signals or a single credible source. Warrants further investigation.

E.g.: Trade press article mentioning exploratory discussions

L

Low

Weak signal, single unverified source or market speculation. Worth monitoring but not actionable as-is.

E.g.: Specialized forum post or indirect mention in general article

Confidence score (0-100)

In addition to the qualitative level (high/medium/low), each deal receives a numeric score from 0 to 100, automatically calculated on 5 criteria:

30 Number of valid sources
25 Source credibility (Reuters, EC, Bloomberg...)
20 Recency (sources < 7 days)
15 Parties identified (named acquirers + targets)
10 Corroboration (already seen in previous scan)

The score is displayed as a colored progress bar on each deal: green (≥ 60), orange (≥ 30), red (< 30).

Confidence threshold (watchlist): When you set a confidence threshold in your watchlist (e.g., "medium"), the AI will only return signals with confidence level equal to or higher than this threshold.

8 Outreach emails

Generate personalized outreach emails for selected actors directly from the Results page. The AI creates a custom hook based on recent news about each actor.

1

Select actors

On the Results page (Actors tab), use the checkboxes to select one or more actors you want to contact.

2

Click "Generate outreach"

A button appears showing the number of selected actors. Click to start email generation.

3

Review and copy

A modal displays generated emails. Each email has a personalized hook with a recent source. Use "Copy" to copy individual emails or "Copy all" for all.

Custom template (optional)

You can define a custom email template in watchlist settings. When enabled, the AI will generate only the hook (first paragraph with recent news) and append your template.

Setup: Go to Watchlists → Edit → Outreach Settings section. Enable "Custom email template" and enter your template text.

9 Frequently asked questions

How many watchlists can I create?

There is no limit. You can create as many watchlists as needed to cover your different monitoring scopes.

What happens when I rerun a scan?

The system performs automatic cross-run deduplication. Deals and actors already identified in previous scans are not duplicated. If an existing signal is detected again, its last seen date is updated.

Can I delete a watchlist?

Yes. Click the trash icon on the watchlist card in the dashboard. Deletion is a soft delete: the watchlist is deactivated but historical results are preserved.

What sources are used?

The platform uses SourceCatalog v1 (50 sources) with pack activation (EU_CORE, NORDICS, FRANCE, GRANTS_EU) plus web search. Screening uses funding/grants filters and Stage1/Stage2, while weak-signals mode keeps pre-announcement maturity gating.

How do I configure the tool for the first time?

On first dashboard load, a configuration form appears. Enter the API base URL (provided by administrator) and the API key. This information is stored locally in your browser.

How does deduplication work?

Each deal and actor is identified by a unique SHA-256 hash, calculated from title, parties, operation type and geography. Company names are normalized (removing legal suffixes like GmbH, Inc, SA). If a deal is detected again in a later scan, it is enriched (merged sources, advanced stage, filled empty fields) instead of duplicated.

What does "Updated X ago" mean on a deal or actor?

This indicator appears when a deal or actor was seen again in a later scan. The first detection date remains displayed, and "Updated" indicates when the last observation occurred. This means the signal is still active and confirmed by recent sources.

Can I filter watchlists by owner?

Yes. On the dashboard, a dropdown above the watchlist list allows filtering by owner. Select a name to see only their watchlists, or "All" to display everything. The owner (purple badge) is also visible on each watchlist card and in the scan history table.

What happens if a scan fails due to insufficient OpenAI funds?

A visual error modal will appear explaining that the scan could not complete due to insufficient funds in the OpenAI account. Contact your administrator to add credits or check the billing page.
